前端项目开发的几个场景

Dec 3, 2015

这里总结我自己的前端项目开发的过程的几个场景:

1. dev (开发场景)

纯正开发模式.

2. release (发布场景)

release 场景下需要做的事情:

  1. 代码编译
  2. 代码压缩
  3. 发布到线上环境

3. testRelease (测试发布场景)

模拟真实线上环境的发布, 不过只是在测试服务器上发布. 这个场景下就可以让测试人员来参黑合测试了.

4. release-preview (发布预览场景)

这是介于 dev 与 release 之间的一步, 这个过程发生在开发这自己的电脑上. 相当与在开发者自己电脑上 simuRelease 一次. 开发者先检查一下有无明显的问题.

通过上面的分类我可以使用更加符合语义的 npm script 了如:

  > npm run dev                     // "写代码去了"

  > npm run release                 // "天啦噜, 终于发布啦"

  > npm run testRelease             // "测试的兄弟们, 我已经部署到内网测试了, 你们看下有什么问题"

  > npm run release-preview         // "终于写完了, 检查看下有没问题先, 等下就可以交给测试了"
